
# 常见问题

## 你何必搞这么复杂？直接用XXMI、WWMI系列工具不行吗？

SSMT不单单是一款工具，它是一种哲学，一种优雅的问题解决方案，如果只是为了制作Mod那直接用XXMI系列工具确实是最好的选择，但SSMT本身是一种艺术，一种思维的哲学，一种优雅的态度，一种对Mod本身的热爱，一种精神上的追求。

SSMT虽然大量学习和借鉴了XXMI、WWMI系列工具，但其最大优点在于其优雅的设计，除此之外没有什么特别的地方，只有当你使用过SSMT后你才能体会到它的精神所在。

技术宅拯救世界(bushi，闲的没事开发着玩儿的，纯玩儿)。

## 你开发SSMT是为了自己制作Mod更方便吗？

并非，我一般懒得制作Mod，开发SSMT纯属为人民服务。

(主要是闲的，总不能天天打游戏吧，大部分游戏都玩腻了)

## 为什么不使用QT和WPF开发？它们都能解决WinUI3的问题

短期来看，3Dmigoto只支持DX11，但是未来可能出现Linux平台以及MacOS、甚至Vision Pro上的类似项目。

SSMT4需要为未来做好准备，这次重构的目光放在了10年甚至20年后，而不仅仅是解决现在的问题，从架构上要做到支援未来。

WPF只能解决Windwos上的问题，无法战未来，战未来这块儿甚至不如WinUI3

QT架构上太过臃肿不适合需要快速迭代的小工具的开发维护，其次就是无法方便的使用Copilot等工具，但是Tauri的架构中开发在VSCode中进行，完美适配需要Copilot AI辅助编程的需求。

我个人部分兴趣内容例如逆向分析、相关工具制作等，使用Rust更方便和优雅，开发网站也是我的目标之一，Rust + Tauri完美适配了我当前的学习技能的所有需求。

此外，如果3Dmigoto未来无法广泛使用，例如由于游戏政策等问题失效，Rust + Tauri架构也可用于其它项目开发，总之学会了肯定不亏的，相比于开发维护SSMT来说，掌握Rust和网页开发知识才是根本目标(因为C/C++/C#/Java/Python我都用过,没有新鲜感了)。


# 为什么前端选择 bun + vite + vue + typescript?

从XiaoLinXiaoZhu那里学过来的，感觉很好很优雅。

Bun速度快，上手快，简单易学

VUE模板多，开发比React简单，上手快

TS则是因为，之前在开发Python写Blender插件的过程中，深刻的感觉到数据类型的重要性，
所以前端肯定不能用JS，肯定得用TS，而且VUE天然支持TS

反正我们只是开发个小工具，架构能用多新就用多新就完了

# Rust 和前端都很难，没人能参与开发咋办？

首先在做这个项目的时候我也是0基础，没有人是天生就会某些东西的，大家都是从0开始学习的。

包括我之前做WinUi3开发，不也是一行一行代码摸索出来的嘛，Rust也不是特别难，我们也用不到Rust难的部分

前端也不需要我们自己写底层，直接拿现有的轮子用就行了。

本质上和之前的WinForm以及WinUI3一样，都是会调别人写好的开箱即用的东西就行了。

从0到1是最难的，从1到90反而简单，从90到100几乎是指数性的难，人类的学习认知模型几乎都是这样的。

别多想，奥里给干就完了。
